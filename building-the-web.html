accessability

Links do not have a discernible name

links should have descriptive text so users with screen readers and assistive technologies understand what this link goes to. As there is an option in screen readers to read just links (not sure)

the problem

the problem comes when the link goes to a social media website (it's just an icon) or it's an article with "read more" or "see full article here" or even just "here"

solution

put all your accessability text in a span in the a and give it the next properties in css

  • display:block;
  • width: 1px;
  • height: 1px;
  • overflow: hidden;
  • position: absolute;
  • top: -10px;

notice you don't need to give the links position: relative; as you actually want all the accessible text to be -10px from the top of the document, so it's visually not there

and this technique is the one descriped by WCAG (my paper say it's from the w3c (not the w3schools) but I remember that it was from WCAG)

Tap targets are not sized appropriately

problem

according to web.dev Targets that are smaller than 48 px by 48 px or closer than 8 px apart fail the audit. When the audit fails, Lighthouse lists the results in a table with three columns:

  • tap target ------------------> The tap target that is inappropriately sized.
  • size ------------------------> The size of the target's bounding rectangle in pixels.
  • Overlapping Target ----> Which other tap targets, if any, are too close.

solution

1.Increase the size of tap targets that are too small. Tap targets that are 48 px by 48 px never fail the audit. If you have elements that shouldn't appear any bigger (for example, icons), try increasing the padding property:

2.Increase the spacing between tap targets that are too close together using properties like margin. There should be at least 8 px between tap targets.

notice that the two pargraphs above are not two options they are two steps which means you should do both of them

as personally I changed the font-size a little bit and the margin but the whole element height wasn't 48px it was actually 26px and I still passed the audits

animation

to add an animation

  1. select the element you want to animate

  2. click on the thunder icon on the right panel at the end of the line of style icon

    as you can see there are two types of triggers: element trigger (like mouse click, mouse hover, mouse over element, scroll into view, and while scrolling in view)

  3. click on the plus icon beside the type you want to show you the optoins and choose the option you want

  4. the follwing dialog will appear

    click on select action drop menu to choose action

    some times there are actions like fade in and fade out, etc... and some other time the only aciton is custom animation

  5. if you choose custom animation (say we chose the custom animation in page scroll in the page trigger type), additional line will appear (scroll animations) showing you the animations you already have (our animation) and offering you to add another animation by clicking the plus icon beside the scoll animtion word

    click it to add a new animation and choose a name for it

  6. now you can choose actions to animate from the plus icon beside the (scroll actions) word

  7. once you choose the action you want two keyframes will appear one at 0% and the other at 100%, you can drag them along the timeline to choose where every keyframe starts

    as long as there is a yellow triangle on a frame it means there's something wrong with and sometimes hovering over the triangle can help you solve the problem

  8. under the timeline you find the other options of the action

when adding an animation make sure that you're using the same unites in both the starting and the ending animation
like in the move animation if you use vh in the x-axis in one of them you'll have to use vh in the x-axis in the other one other wise it keeps showing you the yellow triangle

states

to set styling for the different states of an element click on the small down arrow icon beside the class list of the element in the styling panel and you will see the different states like hover focus and focus keyboard

the difference between focus and focus(keyboard) that focus is applied either if you click the element, tap on touch screens or press tab on the keyboard and it uses the css selector :focus while the focus(keyboard) is only applied when selecting elements with the keyboard tab and uses the css selector :focus-visible I recommend you to read MDN article about it

it can be useful in setting the outline of the elements when tab focused in order to enhance accessability
