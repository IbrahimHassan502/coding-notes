accessability

Links do not have a discernible name

links should have descriptive text so users with screen readers and assistive technologies understand what this link goes to. As there is an option in screen readers to read just links (not sure)

the problem

the problem comes when the link goes to a social media website (it's just an icon) or it's an article with "read more" or "see full article here" or even just "here"

solution

put all your accessability text in a span in the a and give it the next properties in css

  • display:block;
  • width: 1px;
  • height: 1px;
  • overflow: hidden;
  • position: absolute;
  • top: -10px;

notice you don't need to give the links position: relative; as you actually want all the accessible text to be -10px from the top of the document, so it's visually not there

and this technique is the one descriped by WCAG (my paper say it's from the w3c (not the w3schools) but I remember that it was from WCAG)

Tap targets are not sized appropriately

problem

according to web.dev Targets that are smaller than 48 px by 48 px or closer than 8 px apart fail the audit. When the audit fails, Lighthouse lists the results in a table with three columns:

  • tap target ------------------> The tap target that is inappropriately sized.
  • size ------------------------> The size of the target's bounding rectangle in pixels.
  • Overlapping Target ----> Which other tap targets, if any, are too close.

solution

1.Increase the size of tap targets that are too small. Tap targets that are 48 px by 48 px never fail the audit. If you have elements that shouldn't appear any bigger (for example, icons), try increasing the padding property:

2.Increase the spacing between tap targets that are too close together using properties like margin. There should be at least 8 px between tap targets.

notice that the two pargraphs above are not two options they are two steps which means you should do both of them

as personally I changed the font-size a little bit and the margin but the whole element height wasn't 48px it was actually 26px and I still passed the audits

quick stack is powerful grid element where you have more freedom merging cells and doing all kinds of layouts

  • grid cells hold only one element by default to overcome this you can add a dive inside the cell to group multiple elements inside one cell or hold control then drag the each element into the cell
  • to make a cell span more columns or more row simply click and drag the small ball on the right bottom corner of the cell

manually dragging a grid element

if you try to manually drag a grid element from cell to cell you'll find that it doesn't move

to overcome this simply go to the postion section that is under grid child deopdown in the style panel and set the position to manual

notice that when you manually postion an element that doesn't change it's order in the actual DOM so screen readers will still see it in it's original postion before you manually moved it

so basically dense will fill the gap with the next available child as long as the child fits in the gap
