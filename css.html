justify self & justify items work only with grid and don't work with flexbox

The justify-self and justify-items properties are not implemented in flexbox. This is due to the one-dimensional nature of flexbox, and that there may be multiple items along the axis, making it impossible to justify a single item. To align items along the main, inline axis in flexbox you use the justify-content property.
as in the flexbox (on the main axis) children are treated as a group

flexbox VS grid

the main difference between the two is that use felxbox if you want to control the content in one direction (row or column) and use grid if you want to control the content in both directions (row and column).Like the pervious images may illustrate.

another piece of advice to help you decide is to ask your self
Do you want your content to shape your layout ? ==> use grid
Do you want your layout to shape your content ? ==> use grid

finally one piece of adviece: In grid-layout there is no way to change the size of one single items without changin the whole column.

my opionion is what ever gets the job done with less code you go with it even if in your beginnings you write the same layout twice one with grid and one with flex to help you decide it's okay.

I think the key phrase here is ‚Äúdistributing space and aligning content‚Äù. Flexbox is all about taking a bunch of things (which have varying sizes) and fitting them into a container (which itself has a varying size).
  • When people find Flexbox hard and weird, it is often because they are trying to use Flexbox as a grid system ‚Äî trying to take back control over sizing and space distribution. When you do this, Flexbox can seem weird and hard as you are fighting against the very thing that makes it Flexbox, i.e. that inherent flexibility.
    Therefore, patterns which suit flex layout very well are those where we are not so interested in having a pixel-perfect size for each item. We just want those items to display well alongside each other.
Remember that you can also switch layout methods, using Flexbox at one breakpoint and Grid at another.

One of the media features we can now test for with media queries is whether the user's primary input is a touchscreen (pointer) and whether any of the currently detected inputs is a touchscreen (any-pointer). The pointer and span>any-pointer features will return fine or coarse.

  • A fine pointer will be someone using a mouse or trackpad, even if that mouse is connected via Bluetooth to a phone.

custom styles

every custom css you add has a layer to go whether it's in the base, components or utilites or even just adding it to the src style

when and why to use these layers

let's start with the why:

  1. tailwind organizes the classes in layers where base layer comes first then components then utilities where utilities overrides the first two layers and components overrides the base styles so explicitly categorizing your style helps in override and avoiding conflicts
  2. when you add your style in a layer it can be used with tailwind modifiers like hover:, md: and so on
  3. on build time tailwind removes all unused classes so if you just add your style in the src style without putting it in a layer it stayes even if it's unused

for more info visit adding custom styles docs
