images width and height attributes

why to add them

If you don't set the width and height attributes the browser will wait unitl the image is loaded to calculate the dimentions of the image and then make a space for it and it's bad for two reasons:

  • layout shift: the text being suddenly pushed down when the image is loaded which is really annoying
  • increasing the loading time and decreasing the performance of the page as the browser has to calculate the dimentions itself

what happens when you add them ?

the browser takes the width and the height and put them in an aspect-ratio in the user agent so it makes a space for the image without the need of waiting it to download.

note: when you set when dimention with css (width or height) you should make sure you set the other dimention (height or width) to auto.Otherwise the value will still be taken the attriubte you specified before. when you set the other dimention to auto: the browser calculates it based on the aspect ratio it has (from the width and the height attributes) without the need of waiting the image to download

downsides
  • the implementation of the browser detecting the aspect-ratio based on the width and the height attriubtes is still new (around 2020).
    the aspect-ratio css proprety for images itself is still new
  • for responsive images like the code below you can only set width and height attribute to the main image and you can't set them to the images in srcset.
    the best solution so far is to make sure that all of them have the same aspect-ratio
  • it doesn't work width art direction read about it if you want I want brief it here as it's not relavent to me at this time
  • ads: During the ad lifecycle, many points can introduce layout shift:
    • When a site inserts the ad container in the DOM
    • When a site resizes the ad container with first-party code
    • When the ad tag library loads (and resizes the ad container)
    • When the ad fills a container (and resizes if the final ad has a different size)
    and the best solution introduced by web.dev is to pre-reserve a space to the ad by setting dimetinos for an ad container and make sure it is in the page wheather the ad is loaded or not to avoid layout shift when the ad fails to load.
    This solution can have some downsides like the ad being smaller than the container you made which leades to blank spaces.

conclusion: set width and height for img and video elements as it increases performance and make sure when you change one dimention by css you set the other one to auto to allow it to be calculated by the browser and to avoid stretched or squished images.

What are void elements in HTML?

HTML elements which do not have closing tags or do not need to be closed are Void elements. For Example <br />, <img />, <hr />

What is the advantage of collapsing white space?

In HTML, a blank sequence of whitespace characters is treated as a single space character, Because the browser collapses multiple spaces into a single space character and this helps a developer to indent lines of text without worrying about multiple spaces and maintain readability and understandability of HTML codes
